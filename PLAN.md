# 전 세계 지명 확장 계획

## 목표

현재처럼 소수의 정적 배열만 계속 늘리는 방식에서 벗어나, 전 세계의 도시와 자연지명을
줌 단계에 맞춰 최대한 많이 표시하되 화면 혼잡과 초기 로딩 비용을 통제한다.

## 이번 단계에서 반영한 범위

- 중국: 기존 주요 도시 외 35개 추가
- 일본: 기존 주요 도시 외 26개 추가
- 미국: 기존 주요 도시 외 44개 추가
- 상세 도시는 `detailed-cities.js`로 분리
- 인구와 지역 대표성에 따라 `minZoom`을 6~8로 나누고, 상세 줌인 z15까지 유지
- 좌표는 GeoNames `cities15000` 공개 덤프를 기준으로 대조

## 권장 데이터 소스

### 1. GeoNames — 기본 지명 데이터

- 도시, 마을, 산, 호수, 하천, 섬, 사막 등 다양한 feature class를 한 체계로 제공
- 국가별 파일 및 전 세계 덤프를 매일 갱신
- `cities15000`, `cities5000`, `allCountries`, `alternateNamesV2` 활용
- 한글명은 `alternateNamesV2`에서 언어 코드 `ko`를 우선 사용
- 라이선스: CC BY 4.0. 앱과 저장소에 GeoNames 출처 표기가 필요

### 2. Natural Earth — 저줌 우선순위와 지역 대표성

- 국가·주/도·대표 도시·섬·해역 등 세계 축척용 선별에 적합
- `scalerank`, `rank_max`, `rank_min`을 저줌 라벨 우선순위에 사용
- 데이터는 Public Domain

### 3. OpenStreetMap — 고줌 보완 데이터

- 작은 마을, 봉우리, 관광지, 역, 공원 등 가장 풍부한 세부 지명
- 전체 데이터를 앱에 직접 포함하지 않고 지역별 추출 또는 서버 타일 방식으로 사용
- ODbL 출처 표시와 파생 데이터 배포 조건을 설계 단계에서 반영해야 함

## 넣을 지명 종류

### 행정·정착지

1. 국가와 속령
2. 수도
3. 주·도·성 등 1급 행정구역
4. 주요 도시와 광역권 중심
5. 중소 도시, 군청·현청 소재지
6. 작은 마을과 오지의 지역 중심지

### 자연지명

1. 산, 화산, 주요 봉우리
2. 산맥, 고원, 분지
3. 강, 하천, 폭포
4. 호수, 저수지, 습지
5. 바다, 만, 해협, 곶
6. 섬, 군도
7. 사막, 초원, 숲
8. 계곡, 협곡, 동굴
9. 빙하, 빙원

### 문화·교통 지명

1. 국립공원과 세계유산
2. 주요 항구와 공항
3. 유명 유적과 랜드마크
4. 주요 철도역과 산악 고개

## 줌 단계별 표시 기준

| 줌 | 표시 대상 |
| --- | --- |
| z3~4 | 국가, 수도, 대륙급 산맥·해역 |
| z5~6 | 초대형 도시, 1급 행정 중심, 대표 산·강·호수 |
| z7~8 | 주요 도시, 지방 중심지, 유명 자연지명 |
| z9~10 | 중소 도시, 산·섬·공원, 주요 관광지 |
| z11~12 | 작은 도시와 마을, 세부 자연지명 |
| z13~15 | 지역 단위 마을, 봉우리, 계곡, 역·항구·랜드마크 |

인구만으로 순위를 정하지 않는다. 오지의 유일한 지역 중심지, 섬의 대표 도시, 행정 수도,
관광·역사적 중요 지명에는 별도 가중치를 준다.

## 구현 구조

### 1단계 — 생성 스크립트 도입

- `scripts/build-place-labels.mjs` 추가
- GeoNames 덤프와 한글 대체명을 읽어 정규화
- 국가 코드, feature code, 인구, 행정 등급으로 우선순위 계산
- 중복 좌표와 동명이인 처리
- 결과를 지역별 JSON 또는 JS 모듈로 생성

### 2단계 — 지역별 데이터 분할

- `labels/asia/*.json`, `labels/americas/*.json`처럼 대륙 또는 타일 단위로 분리
- 현재 화면 bbox에 필요한 파일만 동적 import/fetch
- 모든 지명을 시작 시 메모리에 넣지 않도록 함

### 3단계 — 충돌 회피

- 화면 좌표 기준 라벨 bounding box 계산
- 우선순위가 높은 라벨부터 배치
- 겹치는 낮은 우선순위 라벨은 숨김
- 수도·현재 위치·사용자가 고정한 라벨은 우선 표시

### 4단계 — 공간 인덱스

- R-tree 또는 줌/타일 키 기반 인덱스 사용
- 현재 지형판과 교차하는 지명만 조회
- 수만~수십만 지명에서도 전체 배열 순회를 피함

### 5단계 — 사용자 설정

- 도시, 자연지명, 행정구역, 관광지 카테고리별 표시 스위치
- 지명 밀도: 적게 / 보통 / 많이 / 최대
- 최소 인구 또는 최소 중요도 선택
- 한글명 우선, 현지명 병기, 영문명 병기 옵션

## 데이터 품질 규칙

- 좌표계는 WGS84 위·경도로 통일
- 한글명이 있으면 한글명, 없으면 현지명 또는 영문명 사용
- 동일 지명이 가까운 좌표에 중복되면 feature 우선순위로 하나만 유지
- 수도·도시·산처럼 종류가 다른 동일 명칭은 유형을 키에 포함
- 정치적·영토 분쟁 지역은 데이터 원문과 별도로 표시 정책을 문서화
- 자동 생성 결과에 원본 ID와 출처를 남겨 수정과 갱신이 가능하게 함

## 성능 목표

- 초기 포함 데이터: 핵심 지명 2,000개 이하
- 지역별 지연 로딩 데이터: 전 세계 도시·자연지명 50,000~200,000개
- 프레임당 전체 지명 순회 금지
- 화면에 실제 생성되는 CanvasTexture 라벨은 약 100~300개 이내
- 라벨 텍스처는 LRU 캐시로 관리하고 화면 밖 장기 미사용 항목은 해제
